Semgrep is a command-line tool. Every capability is exposed through subcommands that share a consistent set of global flags.

## Subcommands

```bash
semgrep [OPTIONS] COMMAND [ARGS]
```

## Version flag

```bash
semgrep --version
```

Prints the installed Semgrep version and exits.

## Global flags

These flags apply to every subcommand.

| Flag | Description |
|------|-------------|
| `-h`, `--help` | Show help and exit. |
| `--version` | Show version and exit. |
| `-v`, `--verbose` | Show more details: extra timing output, skipped targets, and verbose error messages. |
| `-q`, `--quiet` | Suppress all output except findings. |
| `--debug` | Show all internal logging. Implies `--verbose`. |
| `--force-color` / `--no-force-color` | Force color output even when writing to a pipe or file. |
| `--metrics [auto\|on\|off]` | Control anonymized usage metrics. `auto` (default) sends metrics only when using Semgrep Registry rules. Set `SEMGREP_SEND_METRICS` to override. |

## Exit codes

All Semgrep commands exit with a code that reflects the scan outcome.

| Exit code | Meaning |
|-----------|---------|
| `0` | Success, no findings reported. |
| `1` | One or more findings were reported. |
| `2` | A fatal error occurred (internal error, invalid arguments). |
| `3` | One or more target files failed to parse. |
| `4` | One or more rule files failed to parse. |
| `5` | Unparseable YAML. |
| `7` | Missing or invalid API key (`SEMGREP_APP_TOKEN`). |
| `8` | Invalid language specified. |
| `13` | Invalid API key — authentication rejected by the server. |

<Note>
  By default, `semgrep scan` exits with code `0` even when findings are reported. Use `--error` to exit with code `1` when findings exist.
</Note>
