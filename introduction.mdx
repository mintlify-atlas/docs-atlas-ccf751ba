## What is Semgrep?

Semgrep is a fast, open-source static analysis tool that searches code, finds bugs, and enforces secure guardrails and coding standards. Semgrep supports 30+ languages and can run in an IDE, as a pre-commit check, and as part of CI/CD workflows.

Semgrep is **semantic grep for code**. While running `grep "2"` would only match the exact string _2_, Semgrep would match `x = 1; y = x + 1` when searching for _2_. Semgrep rules look like the code you already write — no abstract syntax trees, regex wrestling, or painful DSLs.

## Why Semgrep?

### Rules that look like code

Semgrep rules are written in YAML and use patterns that mirror actual source code. There's no special DSL to learn — if you can write the code, you can write the rule.

```yaml
rules:
  - id: hardcoded-password
    patterns:
      - pattern: password = "$PWD"
      - pattern-not: password = ""
    message: Hardcoded password detected
    severity: ERROR
    languages: [python]
```

### 30+ languages supported

Apex · Bash · C · C++ · C# · Clojure · Dart · Dockerfile · Elixir · HTML · Go · Java · JavaScript · JSX · JSON · Julia · Jsonnet · Kotlin · Lisp · Lua · OCaml · PHP · Python · R · Ruby · Rust · Scala · Scheme · Solidity · Swift · Terraform · TypeScript · TSX · YAML · XML · Generic

### Code is analyzed locally

Semgrep analyzes code locally on your computer or in your build environment: **by default, code is never uploaded**.

### Fast and scalable

Semgrep is designed to run fast at scale. The core analysis engine is written in OCaml for performance, while the CLI and orchestration layer are written in Python.

## Getting started

<Steps>
  <Step title="Install Semgrep">
    Install via Homebrew, pip, or Docker. See the [installation guide](/installation) for details.
    ```bash
    brew install semgrep
    # or
    python3 -m pip install semgrep
    ```
  </Step>
  <Step title="Run your first scan">
    Run Semgrep against your codebase using community rules:
    ```bash
    semgrep --config=auto .
    ```
  </Step>
  <Step title="Explore the results">
    Review findings in your terminal, or use `--output` to save results as JSON or SARIF for integration with your tooling.
  </Step>
  <Step title="Write custom rules">
    Create rules tuned to your codebase. Rules look like the code you already write. See [Writing Rules](/writing-rules/overview) to get started.
  </Step>
</Steps>
