## Before you begin

You need one of the following to install Semgrep:

- **Homebrew** (macOS)
- **Python 3.10+** with pip (macOS, Linux, Windows)
- **Docker** (any platform)

<Steps>
  <Step title="Install Semgrep">
    Choose the installation method that fits your environment.

    <CodeGroup>

    ```bash Homebrew (macOS)
    brew install semgrep
    ```

    ```bash pip (macOS / Linux / WSL)
    python3 -m pip install semgrep
    ```

    ```bash Docker
    docker pull semgrep/semgrep:latest
    ```

    </CodeGroup>

    Verify the installation:

    ```bash
    semgrep --version
    ```

    You should see output like `1.155.0`.
  </Step>

  <Step title="Run your first scan">
    Navigate to the root of any code repository and run:

    ```bash
    semgrep --config=auto .
    ```

    The `--config=auto` flag tells Semgrep to automatically select rules from the
    [Semgrep Registry](https://semgrep.dev/explore) that match the languages
    detected in your project. Semgrep scans locally — **your code is never uploaded**.

    <Tip>
      Running Semgrep for the first time on a large repository may take a minute
      while rules are fetched. Subsequent runs are faster.
    </Tip>
  </Step>

  <Step title="Interpret the output">
    Each finding Semgrep reports follows this format:

    ```
    rule-id
    path/to/file.py:42:8
    ❯❯❱ python.lang.security.audit.formatted-sql-query.formatted-sql-query
          SQL query built with string formatting is vulnerable to SQL injection.

           40 ┆ def get_user(username):
           41 ┆     query = "SELECT * FROM users WHERE name = '%s'" % username
           42 ┆     cursor.execute(query)
    ```

    The fields in a finding are:

    | Field | Description |
    |-------|-------------|
    | Rule ID | The unique identifier of the rule that matched (e.g., `python.lang.security.audit.formatted-sql-query`) |
    | File path | The file where the match was found |
    | Line number | The line (and column) of the match |
    | Message | A human-readable description of the issue |
    | Code snippet | The matched lines with context |

    At the end of the scan, Semgrep prints a summary:

    ```
    Ran 143 rules on 27 files: 3 findings.
    ```
  </Step>

  <Step title="Try a pattern search">
    You can search for specific code patterns without writing a full rule file.
    Use `-e` to pass a pattern directly and `--lang` to specify the language.

    For example, find places in Python code where a value is compared to itself
    (a likely bug):

    ```bash
    semgrep -e '$X == $X' --lang=py path/to/src
    ```

    `$X` is a **metavariable** — it matches any expression. This pattern catches
    code like:

    ```python
    if user.role == user.role:   # always True — likely a copy-paste error
        grant_access()
    ```

    You can use the same approach for other languages:

    ```bash
    # Find hardcoded empty password strings in JavaScript
    semgrep -e 'password = ""' --lang=js src/

    # Find calls to exec() in Ruby
    semgrep -e 'exec(...)' --lang=rb .
    ```
  </Step>

  <Step title="Log in for Pro rules (optional)">
    Logging in gives you access to 600+ high-confidence rules maintained by
    Semgrep's security research team, cross-file and cross-function analysis,
    and Semgrep Supply Chain for dependency scanning.

    ```bash
    semgrep login
    ```

    After logging in, run a full scan with Pro rules:

    ```bash
    semgrep ci
    ```

    <Note>
      Login is optional. You can use Semgrep Community Edition with
      `--config=auto` or any local rule files without an account.
    </Note>
  </Step>
</Steps>
